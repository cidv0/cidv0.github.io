Abiotic Factor Wiki

Agent Workflow Playbook

This page defines the repeatable workflow for any agent contributing to the Abiotic Factor gameplay wiki. Follow these steps to choose a task, research the topic, produce an HTML guide, and keep the roadmap in sync.

← Back to Wiki Overview Site Root

1. Understand the Project Guardrails

  • Spoiler-free mandate: focus on mechanics, systems, and player agency. Avoid revealing scripted story beats, plot twists, or developer-set piece details.
  • Source of truth: treat wiki-data.js as the master list of sections, titles, and descriptions. Update it if you add or remove guides.
  • Draft-first workflow: every new guide starts life as a planning note in a matching *.html.txt file. Convert to HTML only after research is complete.
  • Asset hygiene: keep filenames lowercase with hyphens, use ASCII text, and mirror directory structure when adding new content.

2. Pick Your Next Task

  1. Open Abiotic-Factor/index.html. Scan for red links (missing) or amber links (status unknown). Each corresponds to a yet-to-be-built page.
  2. For any candidate topic, open the associated planner inside the folder (for example survival-mechanics/fatigue-and-sleep.html.txt) and confirm there is no completed .html counterpart.
  3. Claim exactly one topic. If you are coordinating with other agents, note your pick in your task tracker to avoid collisions.

3. Research & Capture Findings

Use the planner file as your research checklist. Update it with citations or bullet notes as you gather data. Recommended sources:

  • In-game verification: screenshots, recorded stats, terminal logs.
  • Official materials: patch notes, developer blogs, Steam announcements.
  • Community testing: forum posts, spreadsheets, guide videos (cite cautiously and verify).

Capture answers to every “Gameplay Question” in the planner. Mark unresolved items for follow-up.

4. Draft the HTML Guide

  1. Duplicate an existing section page for layout reference (for example, use the structure of survival-mechanics/day-1-night-1.html or create a new layout with Tailwind CSS).
  2. Transform planner bullet points into narrative guide sections. Keep headings meaningful, use concise paragraphs and action-oriented callouts.
  3. Cross-link relevant guides using relative paths (e.g., ../item-mechanics/crafting-stations.html).
  4. When the HTML file is complete, remove or archive the corresponding *.html.txt planner.

5. Wire Up Links & Validate

  • Ensure the new file name exactly matches the href defined in wiki-data.js. Update the data file if you introduced new subtopics or renamed an entry.
  • Reload the landing page in a local web server. The link checker should repaint your new page green to confirm it is accessible.
  • Navigate through the section overview (for example survival-mechanics/index.html) and verify the link there resolves correctly.

6. Final Checklist Before Handoff

  • ✅ Planner cleared: the *.html.txt is removed or marked complete.
  • ✅ Links verified: landing grid and section page show the link in green.
  • ✅ Sources cited: embedded callouts or footnotes credit key discoveries (if applicable).
  • ✅ Spoilers avoided: re-read for lore or narrative reveals and trim them out.
  • ✅ Notes committed: include a brief summary in your change log or PR describing the new guide.
